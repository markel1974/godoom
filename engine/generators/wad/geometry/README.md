# Geometry: Robust Constrained Delaunay Triangulation & PSLG Engine

A high-performance, mathematically rigorous 2D computational geometry package designed for compiling concave polygons, complex boundaries with multiple holes, and legacy map formats (e.g., BSP/WAD architectures) into strict manifold meshes.

## Architecture Overview

The engine implements a Constrained Delaunay Triangulation (CDT) pipeline built to withstand the floating-point instabilities and topological degeneracies inherent in arbitrary Planar Straight-Line Graphs (PSLG). It emphasizes memory locality, deterministic geometric resolution, and absolute topological invariants.

### Core Subsystems

1.  **Robust Geometric Predicates**
    Standard IEEE 754 floating-point arithmetic is notoriously unstable for collinearity and co-circularity tests. This engine employs adaptive precision:
    * **Fast-Path:** Analyzes determinants against precomputed machine epsilon error bounds (`errBoundOri`, `errBoundInc`).
    * **Exact Fallback:** Switches transparently to 256-bit arbitrary-precision arithmetic (`math/big`) when numerical truncation threatens topological integrity.
    * **Sign Extraction:** Exact predicates return native determinant signs rather than casted magnitude, preventing silent float64 underflows on infinitesimal non-zero values.

2.  **Topological Pre-processing (PSLG Sanitization)**
    Direct triangulation of unverified constraint edges leads to mesh failure. The `SanitizePSLG` routine guarantees a strictly valid planar graph:
    * **T-Junction Resolution:** Deterministic collinear vertex injection.
    * **Edge Intersection Handling:** Simultaneous, in-place constraint fragmentation. Calculates exact intersections and prevents overlapping bounds.
    * **Degeneracy Purging:** Lexicographic normalization (`EdgeKey`) ensures strict constraint deduplication and filters zero-length segments generated by exact-to-float quantization.

3.  **Triangulation Pipeline**
    * **Bowyer-Watson:** Unconstrained Delaunay generation over the sanitized vertex superset. Includes a deterministic spatial purge pass to eliminate bounding super-triangle contamination before constraint recovery.
    * **Lawson Edge Flipping (`RecoverConstraints`):** Deterministic FIFO edge flipping strictly enforces original topological boundaries, effectively converting the Delaunay mesh into a Constrained Delaunay mesh.
    * **Domain Culling:** Employs the triangle incenter (rather than the centroid) for ray-casting validation against the complex polygon boundary, maximizing numerical safety margin for sliver triangles.

4.  **Complex Topology Management**
    * **Hole Bridging (`BridgeHoles`):** Merges interior disjoint holes into a single contiguous boundary. Designed for zero-allocation performance via exact capacity pre-calculation and in-place memory shifting.
    * **Fast Rejection:** Extensive use of branch-predictor-friendly, inlined Axis-Aligned Bounding Box (AABB) checks for ray-casting and line-segment intersections.

## Memory & GC Considerations

While the complex geometry transformations are designed to minimize heap fragmentation (e.g., pre-allocated shift operations in `BridgeHoles`), extreme degenerate topologies relying heavily on the exact predicate fallbacks (`Orient2DExact`, `InCircle2DExact`) will instantiate `big.Float` objects. Future iterations may target `sync.Pool` integration or static array implementations for Shewchuk predicates to further minimize GC pressure during the map compilation phase.

## API Surface Structure

* `Point`, `Edge`, `Triangle`: Primitive layout structs.
* `Polygon`: Primary domain structure handling core loops, signed areas, and CDT orchestration (`Triangulate`).
* `ComplexPolygon`: Meta-structure managing non-manifold definitions (Outer bounds + Holes) and topological bridging.
* `geometry.go`: Root level mathematical and mathematical pure logic (Determinants, Exact Algebra, PSLG constraints recovery).
